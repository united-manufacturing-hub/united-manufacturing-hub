Migration-window exemption sanity check
---------------------------------------

The exampleparent registry entry is flagged migrationWindow=true; its
mirror is allowed to return principled nil ONLY (`{ _ = snap; return nil }`).
The exemption is itself tested: if a developer were to "fix" the mirror by
returning `[]ChildSpec{}` instead of `nil`, the supervisor's P2.4
discriminator would treat that as "I am a parent and I want zero children
right now" → silently despawn child-0/child-1/child-2 (the teaching
children documented in worker_test.go). The test catches that scenario
because `[]ChildSpec{}` is not in the allow-list of principled-nil shapes
in isPrincipledNilBody.

Conclusion
----------
The mirror byte-equivalence test correctly:
  1. PASSES on the unmodified P2.4 tip (application mirror byte-
     equivalent to canonical; exampleparent mirror returns principled
     nil; communicator/transport skipped because they have no mirror).
  2. FAILS with file paths + canonical-vs-mirror diff when one of the
     two files diverges on a load-bearing token.
  3. Returns to PASS after the divergence is reverted.

The test ships at P2.4 alongside the supervisor cutover. It will remain
in effect until the state-package mirrors are retired (i.e. when the
parent → state import cycle is broken via a different mechanism). At
that point, the registry shrinks but the test stays in place to defend
against any reintroduced mirror.
